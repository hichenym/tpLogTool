# 快速开始指南

## 适用范围

本指南适用于当前版本的 `tpLogTool`，帮助首次使用者完成安装、配置并体验设备、调试和命令功能。

## 首次使用前

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 启动程序

```bash
python run.py
```

## 首次必须配置的账号

程序当前有三类账号：

- 运维账号：设备查询、设备密码获取、状态类接口
- Seetong 账号：调试页和命令页远程连接设备
- 固件账号：固件页面相关功能

打开方式：

1. 启动程序
2. 点击右上角设置按钮
3. 在账号配置页填写对应账号与密码
4. 可先点击验证
5. 点击保存

说明：

- 单个平台的账号和密码必须同时填写或同时为空
- 调试页与命令页必须同时具备运维账号和 Seetong 账号
- 固件页只依赖固件账号

## 配置保存位置

程序配置保存在 Windows 注册表：

```text
HKEY_CURRENT_USER\Software\TPQueryTool
```

其中会保存：

- 账号密码
- 下载路径
- 调试页最近 SN
- 调试页快捷命令
- 命令页 SN 列表
- 命令页命令列表
- 主题

## 页面入口

当前页面顺序：

1. 设备
2. 调试
3. 命令
4. 固件
5. 记录
6. GIT

## 常见使用场景

### 场景 1：设备查询

1. 打开“设备”页
2. 输入设备 SN 或设备 ID
3. 点击查询
4. 查看设备状态、版本、密码等结果

### 场景 2：进入调试页连接设备

方式一：

1. 打开“调试”页
2. 输入设备 SN
3. 检查下载目录
4. 点击连接按钮

方式二：

1. 在“设备”页查询到目标设备
2. 右键设备行
3. 点击“连接调试”
4. 程序会自动跳转到调试页并发起连接

连接成功后：

- 程序会自动发送 `syscmd start`
- 可以直接在交互区底部输入命令并回车发送
- 也可以使用下方命令输入区、快捷命令区

### 场景 3：在调试页下载文件

1. 连接设备成功
2. 输入 `GetSystemCfg /目标路径`
3. 回车发送
4. 文件会下载到“下载目录/SN/”下

### 场景 4：批量执行命令或拉日志

1. 打开“命令”页
2. 在 SN 列表中一行一个输入设备 SN
3. 在命令执行列表中一行一条输入命令
4. 选择下载目录
5. 点击运行按钮

执行结果会在下方表格中展示：

- 每台设备的状态
- 每个文件的下载结果
- 每条命令的执行详情

### 场景 5：固件管理

1. 打开“固件”页
2. 根据条件筛选固件
3. 双击或右键进行编辑

### 场景 6：错误记录查询

1. 打开“记录”页
2. 至少填写一个筛选条件
3. 点击查询
4. 双击结果中的 `设备SN` 查看设备详情

## 调试页使用说明

调试页主要能力：

- 单设备远程命令交互
- 时间戳开关
- 快捷命令保存
- 快捷命令拖拽排序
- 下载目录双击打开
- 连接中可取消
- 已连接时支持注销

提示：

- 快捷命令最多 50 条
- 交互区右键可清空内容
- 命令输入支持上下键切换历史

## 命令页使用说明

命令页主要能力：

- 批量设备并发执行命令
- `GetSystemCfg` 文件批量下载
- 运行中取消
- 执行完成统计
- 结果表格双击复制

提示：

- 默认最多并发 20 台设备
- 下载文件按 `下载目录/SN/文件名` 保存
- 如果设备离线、唤醒失败、文件不存在，会在状态和详情中显示

## 未配置账号时的提示

当使用依赖账号的页面时，如果账号未配置，程序会弹出提示框并可直接跳转到对应配置页。

常见情况：

- 设备页缺运维账号
- 调试页缺运维账号或 Seetong 账号
- 命令页缺运维账号或 Seetong 账号
- 固件页缺固件账号

## 注意事项

1. 首次使用必须先配置账号
2. 调试与命令功能依赖网络和 Seetong 云端鉴权
3. 账号密码当前以 `Base64` 编码存储在注册表中，不是强加密
4. `GetSystemCfg` 下载文件较大时，界面会按进度更新，不需要重复点击
5. 打包版本运行调试功能时会依赖 `query_tool/dll` 中的动态库
